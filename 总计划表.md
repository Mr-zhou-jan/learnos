## 一、产品概述

### 1.1 一句话描述
LearnOS 是面向大学生的 AI 学习操作系统，用费曼学习法 + 苏格拉底提问 + 艾宾浩斯遗忘曲线，让用户从"被动看视频"变成"主动思考输出"。

### 1.2 核心价值主张
- **问题：** 大学生看视频课效率极低——看懂了不等于学会了，缺乏反馈和复习机制
- **现有方案：** 网课平台（被动观看）、刷题App（机械练习）、ChatGPT（聊天式，无结构化教学）
- **我们的方案：** AI导师通过费曼训练室强制学生输出 → AI检测知识漏洞 → 针对性强化 → 智能复习

## 二、用户痛点分析

### 2.1 目标用户画像
| 用户类型 | 特征 | 核心需求 | 使用场景 |
|---------|------|---------|---------|
| 工科大学生 | 课程难、自控力弱、考前突击 | 理解抽象概念、系统复习 | 期末备考 |
| 考研学生 | 科目多、需要长期记忆 | 高效复习、薄弱点定位 | 日常备考 |
| 英语学习者 | 碎片化、缺乏输出环境 | 口语练习、语法纠错 | 日常学习 |

### 2.2 痛点优先级
| 优先级 | 痛点描述 | 当前替代方案 | 解决后价值 |
|--------|---------|-------------|-----------|
| P0 | 看视频课被动输入，缺乏主动输出 | 做笔记/刷题（无反馈） | 学习效率 3x |
| P0 | 不知道自己的知识漏洞在哪 | 考完试才知道 | 精准定位薄弱点 |
| P1 | 学完就忘，没有科学复习计划 | 手动做Anki卡片 | 自动复习 |
| P1 | 遇到问题不知道问谁 | 百度/群里问 | AI即时引导 |
| P2 | 缺乏学习动力和成就感 | 无 | 游戏化激励 |

## 三、产品逻辑

### 3.1 核心理念闭环
```
被动观看 → 主动思考 → 主动输出 → AI检测漏洞 → 针对性强化 → 长期记忆
```

### 3.2 四大教学理论
| 理论 | 流程 | 实现模块 |
|------|------|---------|
| 费曼学习法 | AI讲解→学生复述→AI分析漏洞→重讲→再复述 | 费曼训练室 |
| 苏格拉底教学法 | AI连续提问引导思考，不给直接答案 | AI导师 |
| 艾宾浩斯遗忘曲线 | 1/3/7/15/30天自动推送复习 | 智能复习系统 |
| 主动回忆 | 禁止纯阅读，必须回答问题/解释概念 | 全平台强制交互 |

### 3.3 核心功能模块
| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| AI导师 | 知识讲解/提问互动/答案分析/多模型 | P0 |
| 知识图谱 | 课程知识树拆分/节点状态记录 | P0 |
| 掌握度系统 | 0-100分/多维度计算/实时更新 | P0 |
| **学习驾驶舱** | 学了什么/哪不会/为什么不会/下一步学什么 | P0 |
| 费曼训练室 | 复述→评分→反馈→再复述循环 | P1 |
| 智能复习系统 | 日/周/月任务/遗忘风险排序 | P1 |
| 游戏化成长 | XP/等级/成就/连续签到 | P2 |
| 考试模式 | AI出题/自动评分/薄弱点分析 | P2 |

### 3.4 MVP首批课程
英语(CET4/6) · C++ · 大学物理 · 工程力学 · 互换性与测量技术 · 金融投资

### 6.1 核心表

|----|---------|------|

| review_tasks | user_id, node_id, next_review_time, interval, ease | 复习任务 |

### 6.2 Prisma Schema（核心）
```prisma
model User {
  id        String   @id @default(uuid())
  email     String   @unique
  name      String?
  xp        Int      @default(0)
  level     Int      @default(1)
  streak    Int      @default(0)
  masteryScores  MasteryScore[]
  learningRecords LearningRecord[]
  reviewTasks     ReviewTask[]
}

model KnowledgeNode {
  id        String   @id @default(uuid())
  courseId  String
  parentId  String?
  title     String
  children  KnowledgeNode[] @relation("KnowledgeTree")
  parent    KnowledgeNode?  @relation("KnowledgeTree", fields: [parentId], references: [id])
  masteryScores MasteryScore[]
}

model MasteryScore {
  id        String   @id @default(uuid())
  userId    String
  nodeId    String
  score     Float    @default(0)  // 0-100
  accuracy  Float?
  errorRate Float?
  @@unique([userId, nodeId])
}
```
