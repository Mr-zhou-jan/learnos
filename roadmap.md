# LearnOS 路线图

## MVP：完成可用闭环
目标：让用户完成“选课/视频链接→知识提取→诊断→计划→训练”的一次完整学习体验。

- 修稳定入口流程和构建问题。
- 让计划生成按学科选择训练方法。
- 训练中心展示当前课程的今日训练路径。
- 保留本地存储作为 MVP 状态层。
- 成功标准：无 API Key 时有可用兜底，有 API Key 时能生成个性化计划，`npm run build` 通过。

## V2：学习效果闭环
目标：把一次性计划变成可持续学习系统。

- 接入用户登录和数据库持久化。
- 保存每次诊断、练习、复习、掌握度变化。
- 每个训练法提供专属交互页，例如 Debug 挑战、财经案例追问、英语图文阅读。
- 用 FSRS（Free Spaced Repetition Scheduler，间隔重复算法）生成复习任务。
- 成功标准：用户能看到历史曲线、薄弱点变化和复习队列。

## V3：商业化和规模化
目标：形成可收费的 AI 学习操作系统。

- 支持课程包、班级、教师端和机构空间。
- 支持多模型路由、队列任务、长视频异步处理。
- 支持知识库版本管理和多人共享课程。
- 增加订阅、学习报告导出、考试冲刺包。
- 成功标准：付费用户能稳定使用长视频解析、个性化训练和复盘报告。
