# LearnOS 产品提升方案 v2.0

> 基于2026-06-14 深度体验评审，以AI产品经理视角制定
> 核心诊断：**功能多而不精，缺乏留存钩子，无云端存储**

---

## 一、产品定位修正

**当前**：AI驱动的主动学习平台（太泛，用户不知道你是什么）
**建议**：**AI四六级备考教练** — 诊断弱点 → 每日计划 → 逐项攻克 → 考前模拟

> 知识图谱、费曼训练室等非核心模块从侧边栏移除，收到二级页面。

---

## 二、分阶段提升计划

### 🔴 Phase 1：留存地基（2-3天）— 不做完，其他都没意义

| # | 任务 | 当前 | 目标 | 工作量 |
|---|------|------|------|--------|
| 1.1 | **云端存储** | 全部 localStorage | Supabase 同步用户数据（进度/错题/单词/设置），换设备不丢失 | 4h |
| 1.2 | **今日学习路径** | 驾驶舱堆数据 | 清晰卡片流：① 今日单词N个 → ② 阅读1篇 → ③ 听力1组 → ④ 复习错题。每完成一项自动勾选 | 3h |
| 1.3 | **侧边栏瘦身** | 29个链接 | 只保留：今日任务 / 英语训练 / 错题集 / AI教练。其余收入"更多"折叠 | 1h |
| 1.4 | **诊断后引导** | 跳到空荡荡的/plan | 诊断完成 → 自动生成计划 → 直接进"今日任务"，展示第一个待办 | 2h |

**验收**：换浏览器登录数据全在 · 打开3秒知道今天该干嘛 · 做完一题有"下一项"指引

---

### 🟡 Phase 2：体验打磨（3-5天）— 核心模块做到90分

| # | 任务 | 当前 | 目标 | 工作量 |
|---|------|------|------|--------|
| 2.1 | **整卷秒出** | 生成等30秒 | 进入页面预生成 + 缓存，用户点击3秒内出卷 | 3h |
| 2.2 | **真人语音** | 浏览器机械TTS | Edge-TTS / OpenAI TTS 生成 mp3 缓存，听感像真考 | 4h |
| 2.3 | **移动端适配** | 桌面专用 | 侧边栏→底部TabBar，卡片全宽，按钮≥44px触控标准 | 6h |
| 2.4 | **自定义词库** | 只能背全量词库 | 从错题导入 / 手动添加 / 按高频中频低频筛选 | 3h |
| 2.5 | **流式AI反馈** | 提交后干等 | SSE流式输出批改内容，边生成边显示 | 3h |
| 2.6 | **空状态&错误处理** | 空白页/无意义报错 | 每个空状态配插图和引导文案，报错告知下一步怎么做 | 2h |

**验收**：手机完整可用 · 听力像真人 · 出卷3秒 · 任何操作失败有友好提示

---

### 🟢 Phase 3：增长钩子（3-4天）— 让用户愿意每天回来

| # | 任务 | 当前 | 目标 | 工作量 |
|---|------|------|------|--------|
| 3.1 | **全局打卡+热力图** | 仅词汇有 | 每日完成≥2个模块即打卡，首页展示GitHub式打卡热力图 | 3h |
| 3.2 | **学习周报** | 无 | 每周生成：学习时长、正确率趋势、薄弱模块、下周建议，可分享 | 4h |
| 3.3 | **考前冲刺** | 无 | 设考试日 → 倒计时 → 自动每日推送优先级最高任务 | 3h |
| 3.4 | **错题智能重做** | 错题集只能看 | 错题按艾宾浩斯间隔推送，做对3次自动移出 | 3h |
| 3.5 | **PWA离线** | 纯在线 | Service Worker缓存核心页面+词库，断网也能背单词 | 2h |

**验收**：连续7天不思考"今天学什么" · 能看到进步曲线 · 断网能背单词

---

## 三、优先级矩阵

```
影响力高  │ 1.1 云端存储     │ 2.1 考试秒出
          │ 1.2 今日路径     │ 2.4 自定义词库
          │ 1.4 诊断引导     │ 3.1 打卡热力图
          │                 │
          │ 1.3 侧边栏瘦身   │ 3.5 PWA离线
          │ 2.6 空状态优化   │ 3.3 考前冲刺
影响力低  │ 2.5 流式反馈     │ 2.2 真人TTS
          └──────────────────┴────────────────
            低工作量              高工作量
```

**执行顺序**：左上 → 右上 → 右下（左下锦上添花，有空再做）

---

## 四、关键指标

| 指标 | 当前推测 | Phase 1 后目标 | Phase 3 后目标 |
|------|---------|---------------|---------------|
| 次日留存 | ~5% | > 25% | > 40% |
| 单次会话时长 | ~8min | > 15min | > 25min |
| 整卷生成成功率 | ~70% | > 95% | > 99% |
| 移动端可用 | 0% | — | 100% |
| 7日连续打卡率 | ~2% | > 15% | > 35% |

---

## 五、不建议现在做的事

- ❌ 多学科扩展 — 英语都没到90分
- ❌ 社交/排行榜 — 没用户量做社交是空城
- ❌ 会员付费 — 留存没解决，收费赶客
- ❌ 知识图谱可视化 — 价值不明确，先验证核心流程
